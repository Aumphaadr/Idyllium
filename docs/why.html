Шаблон программы

C# требует большого объёма шаблонного синтаксиса даже для простого вывода:

using System;

class Program {
    static void Main(string[] args) {
        Console.WriteLine("Hello, World!");
    }
}

В Idyllium точка входа проста и читаема — это просто main() без аргументов и типов:

use console;

main() {
    console.write("Hello, World!\n");
}

Типы данных

C++ предлагает несколько десятков способов объявления чисел:

int a = 10;
long b = 123456789;
unsigned short c = 42;
float f = 3.14f;
double d = 2.71828;
char ch = 'A';

А в Idyllium всё просто и строго. Только нужные типы, без "усечённых" и "расширенных" вариантов:

int a = 10;
float b = 3.14;
char c = 'A';
string name = "Alice";
bool flag = true;

У C++ — сложность, перегруз. У Idyllium — чистота и логика с первого урока.

Ввод данных

Python путает детей тем, что input() возвращает строку, и число нужно приводить:

a = int(input("Введите число: "))

Java заставляет писать объёмный код с объектами и сканером:

Scanner sc = new Scanner(System.in);
int a = sc.nextInt();

Idyllium использует предсказуемую, единообразную систему:

int a = console.get_int();
float b = console.get_float();
string name = console.get_string();

Каждая функция get_тип() — понятна и объяснима.

Арифметика

C++: деление двух целых чисел — целочисленное, без предупреждения:

int a = 7, b = 2;
float x = a / b; // Результат: 3, а не 3.5!

Idyllium: деление через / всегда даёт float:

int a = 7;
int b = 2;
float result = a / b;  // Результат: 3.5

Idyllium также предоставляет div(a, b) и mod(a, b) для явной работы с целыми числами.

Преобразования типов

C++ предлагает путаницу из функций и классов:

int a = std::stoi("42"); // "Стой", блин. Прекрасное название!
string s = std::to_string(3.14);
int b = atoi(s.c_str());

Какая из них безопасна? Когда использовать .c_str()? Почему atoi не бросает ошибки?

Idyllium предлагает ясные и одинаковые функции:

int a = to_int("42");
float f = to_float("3.14");
string s = to_string(10);

Названия говорят сами за себя. Преобразования всегда явные. Ошибки — понятные.

Конкатенация строк

C++ иногда позволяет сложить строку и число, что может вести к ошибкам и недоумению:

std::string s = "value = " + 2; // Получится "lue = ". Шта?

JavaScript вовсе превращает всё в строку:

let s = "X = " + 42 + "!"; // "X = 42!"

Idyllium разрешает только осмысленные операции:

string s = "X = " + to_string(42); // Явное преобразование
string a = "A" + '!'; // Строка + символ — допустимо

Никакой автоматической каши из типов. Только логика и контроль.

Условный оператор

Python использует слово elif, которое нельзя встретить больше нигде:

if x > 0:
    print("положительное")
elif x == 0:
    print("ноль")
else:
    print("отрицательное")

Idyllium использует классический else if — универсальный и переносимый:

if (x > 0)
{
    console.write("положительное\n");
}
else if (x == 0)
{
    console.write("ноль\n");
}
else
{
    console.write("отрицательное\n");
}

Без "питоновских" изобретений. Только проверенная логика.

Цикл for

Python скрывает детали итерации в range(), который может работать по-разному:

for i in range(5):        # от 0 до 4
for i in range(2, 5):     # от 2 до 4
for i in range(10, 0, -1): # обратный ход

Idyllium показывает полную структуру цикла, как в C++:

for (int i = 0; i < 5; i = i + 1) {
    console.write(i, "\n");
}

Нет "магических" трёх аргументов и скрытых шагов — всё под контролем.

Случайные числа

Python требует разных функций для целых и дробных:

import random
a = random.randint(1, 10)
b = random.uniform(0.0, 1.0) # почему uniform, а не randfloat?

C++ заставляет использовать rand(), srand() и %:

int r = rand() % 10 + 1; // выглядит ужасно

Idyllium объединяет всё в стройной библиотеке random:

use random;
int a = random.create_int(1, 10);
float b = random.create_float(0.0, 1.0);
char c = random.choose_from("ABCD");

Одна библиотека. Чёткие функции. Всё предсказуемо.

Массивы

C++: массивы не проверяют выход за границы:

int arr[3] = {1, 2, 3};
int x = arr[5]; // Поведение не определено, может даже не упасть

Idyllium: выход за границы приводит к ошибке выполнения:

array<int, 3> arr;
arr[0] = 1;
arr[1] = 2;
arr[2] = 3;
int x = arr[5]; // Runtime error

Безопасность важнее гибкости. Ошибки — явные, а не скрытые.

Динамические массивы

C++: vector — совсем не как обычный массив, с перегруженным синтаксисом:

#include <vector>
std::vector<int> v;
v.push_back(5);
v.erase(v.begin() + 0); // Кто придумал так удалять ячейки? За что?

Python позволяет положить всё подряд в один список:

a = [5, "рыба", True, ["веселье", 42 , "хаос"], 17] # данные валяются как попало

Idyllium сохраняет типовую чистоту и читабельность:

dyn_array<int> t;
t.add(24);
t.add(15);
t.add(38);
t.add(65);
t.add("ъуъ"); // не-не-не, с нами такое не прокатит!
t.remove_at(0); // удалится 24

Чёткая типизация, понятные методы, единый стиль — без сюрпризов.

Использование библиотек

C++ пугает детей конструкциями вроде:

#include <iostream>
using namespace std;
std::cout << "Hello" << std::endl; // можно я пойду домой?

А что такое std? Что внутри iostream? Почему одни функции требуют std::, а другие — нет?

Python предлагает сокращать названия, теряя читаемость:

import random as r
a = r.randint(1, 10)

Idyllium использует строгое и единообразное именование:

use random;
int x = random.create_int(1, 10);

Каждая команда вызывается по схеме библиотека.команда() — как в хорошей архитектуре.

Функции

Python не различает void-функции и возвращающие — без аннотаций:

def show(x):
    print(x)

Чем она отличается от:

def square(x):
    return x * x

Только содержимым, но не сигнатурой. Это мешает понимать структуру программы.

Idyllium требует указания возвращаемого типа:

void function show(int x)
{
    console.write(x, "\n");
}

int function square(int x)
{
    return x * x;
}

Благодаря этому студент сразу понимает: что функция делает, и что она возвращает.

int в роли bool

C++ позволяет использовать числа напрямую в условиях:

int a = 6;
if (a) {
    std::cout << "Выполнится";
}

Почему 6 — истина? Почему 0 — ложь? Почему никто не говорит, что это bool?

Idyllium запрещает использовать int как bool. Тип условия должен быть строго логическим:

int a = 6;
if (a) { // Ошибка: тип int не может быть условием }

if (a != 0) { // Явная проверка
    console.write("Выполнится\n");
}

Idyllium не делает догадок за программиста. Всё должно быть выражено явно.

Ошибка = вместо ==

C++ не остановит вас, если вы случайно напишете присваивание вместо сравнения:

int a = 5;
if (a = 0) { // a присвоится 0, условие = false
    std::cout << "Никогда не выполнится";
}

А если написать if (a = 42)? Всё выполнится, но a будет неожиданно изменено!

А Python по-честному сообщает об ошибке:

if x = 5: # SyntaxError: invalid syntax. Maybe you meant '==' instead of '='?

Idyllium тоже выдаёт ошибку: = — это только присваивание. В условии должно быть ==.

int a = 5;
if (a = 0) { // Ошибка компиляции: условие не может быть выражением присваивания }

Idyllium подсказывает: Ты хотел сравнить? Напиши ==. Детская ошибка не станет взрослой проблемой.
