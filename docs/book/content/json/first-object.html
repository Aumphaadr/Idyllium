Первый JSON-объект

JSON-объект похож на коробку с подписанными ячейками. У каждой ячейки есть ключ и значение. Ключ отвечает на вопрос «как это называется?», а значение отвечает на вопрос «что там лежит?».

В тексте JSON ключи всегда пишутся в двойных кавычках. Значения пишутся по своему типу: строка — в кавычках, число — без кавычек, true и false — тоже без кавычек.

Здесь "name", "age" и "is_admin" — ключи. А "Liam", 12 и false — значения. Если читать такой JSON глазами, получается почти обычная анкета: имя — Liam, возраст — 12, администратор — нет.

Создадим объект в Idyllium

Тип json.Object означает JSON-объект. А json.Value означает «любое JSON-значение»: строку, число, булево значение, массив, другой объект или null.

{"name":"Liam","age":12,"is_admin":false}

Компактно и красиво

Метод to_json() создаёт компактную строку без лишних пробелов. Для файла с сохранением чаще приятнее использовать to_pretty_json(): он добавляет переносы строк и отступы.

Ключи не повторяются

В одном объекте ключи должны быть уникальными. Два одинаковых ключа — это как две папки с одинаковым названием в одной папке: непонятно, какую именно открывать.

main.idyl:6: runtime error: json object already has key 'name'

add(), set() и has()

Метод add() добавляет новый ключ. Метод set() изменяет уже существующий ключ. А has() помогает проверить, есть ли такой ключ в объекте.
