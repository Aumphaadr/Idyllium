Файлу понадобился дом

Пока файлов мало, их можно складывать рядом с main.idyl. Но у игры быстро появляются сохранения, картинки, настройки и таблицы уровней. Если свалить всё в одну кучу, найти нужный файл станет труднее, чем нужный носок в ящике после очень бодрой стирки.

Попытка сразу записать файл в несуществующую папку закончится ошибкой:

main.idyl:4: runtime error: file.open() cannot open 'saves/player.txt' for writing: directory does not exist

Файл player.txt создать можно, но сначала программе нужна папка saves.

Создаём папку

Метод create_directory() создаёт одну папку. Её родитель уже должен существовать. Поэтому создать game/saves/players одним обычным вызовом не получится: сначала нет даже папки game.

Что находится по этому пути?

Метод exists() отвечает только на вопрос «что-нибудь здесь есть?». Иногда этого недостаточно: файл нельзя открыть как папку, а в папку нельзя записывать строки как в файл.

true true false

А list_directory() возвращает динамический массив с именами файлов и папок, лежащих непосредственно внутри указанной папки. Имена отсортированы, чтобы один и тот же проект не выдавал случайный порядок на разных компьютерах.

Копирование и переименование

copy() создаёт копию файла или целой папки вместе с её содержимым. rename() меняет путь существующего объекта: этим методом можно и переименовать, и перенести его в другую папку.

Оба метода откажутся работать, если путь назначения уже занят. Это защита от опечатки: программа не должна молча уничтожать старый файл только потому, что ученик случайно указал его имя.

Удаление без неприятных сюрпризов

remove() удаляет файл или пустую папку. Если папка не пустая, обычный вызов остановится с runtime error. Так язык даёт возможность ещё раз проверить путь перед удалением целого дерева.

Поток ещё открыт?

У входных и выходных потоков есть readonly-свойство is_open. Оно равно true до вызова close() и false после него.
