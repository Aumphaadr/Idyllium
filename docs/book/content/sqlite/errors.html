Сначала коротко повторим правила

База открывается через sqlite.open(), один запрос выполняется через execute(), а параметры связываются через bind(). Результат начинается перед первой строкой, поэтому до методов чтения нужен успешный next(). Открытые объекты закрываются методами close().

Теперь намеренно нарушим несколько правил. Такие примеры нужны не для копирования в проект, а для знакомства с сообщениями об ошибках.

Забыли привязать параметр

main.idyl:12: runtime error: sqlite statement has unbound parameter ':level'

В SQL есть параметры :name и :level, но значение получил только первый. Idyllium не превращает забытый параметр в NULL молча: ошибка прямо называет отсутствующий :level.

Выбрали неверный метод чтения

main.idyl:11: runtime error: sqlite column 'name' is text, expected int

Колонка name содержит текст, а программа потребовала целое число через get_int(). Нужно заменить его на get_string() и тип переменной на string.

Обратились к несуществующей колонке

main.idyl:11: runtime error: sqlite result has no column 'score'

Даже если колонка score существует в таблице, её нужно указать в текущем SELECT. Здесь запрос вернул только name, поэтому у результата нет колонки score.

Забыли next()

Новый результат находится перед первой строкой. Getter разрешён только после того, как next() вернул true. Если он вернул false, строк нет и читать тоже нечего.

Попытались использовать закрытый объект

После close() объект больше не связан с ресурсом. Это относится к sqlite.Database, sqlite.Statement и sqlite.Result. Проверить состояние можно через свойство is_open, но обычно проще организовать программу так, чтобы код после close() к объекту уже не обращался.

SQL тоже проверяет свои правила

SQLite отдельно сообщит о несуществующей таблице, ошибочном синтаксисе SQL, повторяющемся уникальном значении и попытке записать NULL в колонку NOT NULL. Сообщение всё равно будет оформлено как обычная runtime error Idyllium и укажет строку программы, на которой был выполнен запрос.

Читайте сообщение слева направо: файл и строка → вид ошибки → объект SQLite → конкретная причина. Как правило, в одной строке уже написано, какой параметр, колонка или тип привёл к проблеме.
