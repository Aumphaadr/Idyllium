Три игрока — три одинаковых запроса?

Если данные уже лежат в массивах, можно создать отдельный подготовленный запрос для каждого игрока. Но текст запроса от игрока к игроку не меняется: меняются только значения параметров. Значит, один sqlite.Statement можно использовать много раз.

Заполняем запрос в цикле

Три выполнения одного Statement
namelevel
Liam7
Mira12
Nika4

Метод execute() не стирает привязанные значения: это позволяет заменить только один параметр и сохранить остальные. Но здесь каждая строка является независимым набором данных, поэтому в начале итерации вызывается clear_bindings().

Зачем очищать привязки?

Представим, что во второй итерации программист случайно забыл привязать level. Без очистки запрос молча использовал бы прошлый уровень. После clear_bindings() забытый параметр останется пустым, и Idyllium остановит программу с понятной ошибкой вместо тихой порчи данных.

main.idyl:4: runtime error: sqlite statement has unbound parameter ':level'

Повторный bind заменяет значение

Если привязать одно имя дважды, в запрос отправится последнее значение. Это бывает удобно, когда почти все параметры одинаковы.
