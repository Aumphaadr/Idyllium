Пустая строка — тоже значение

У игрока может не быть прозвища. Можно сохранить пустую строку, но она означает «прозвище известно и состоит из нуля символов». Число 0 тоже является настоящим числом, а false — настоящим булевым значением.

Для случая «значение отсутствует или неизвестно» в SQL существует NULL. В Idyllium ему соответствует языковое значение null.

Мы уже встречали его в разделе про JSON: json.Value можно сравнивать с null и проверять методом is_null(). Обычные int, string и другие типы от этого не становятся «наполовину пустыми». Значение null принимают только специальные типы, для которых отсутствие данных действительно имеет смысл.

Какие колонки разрешают null?

Имя помечено как NOT NULL, поэтому оно обязательно. У колонки nickname такого ограничения нет: она может хранить текст или NULL.

Мы впервые добавили к типу колонки SQL-ограничение. Запись name TEXT NOT NULL читается так: «колонка name хранит текст и не разрешает неизвестное значение».

Вызов get_string("nickname") для SQL NULL завершится понятной ошибкой: неизвестное прозвище нельзя превратить в строку. Сначала значение проверяется.

sqlite.Value: значение неизвестного типа

Метод get() возвращает универсальное sqlite.Value. Оно полезно, когда значение может оказаться null или его тип заранее неизвестен.

Также доступны проверки is_null(), is_int(), is_float() и is_string(), а после проверки — преобразования to_int(), to_int64(), to_float(), to_string() и to_bool().
