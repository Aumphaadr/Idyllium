Когда SQL становится тесно в строках

Пока таблица была маленькой, её устройство помещалось в одну строку Idyllium. В настоящей базе могут быть десятки таблиц, индексы и начальные данные. Склеивать такой SQL из множества строк неудобно: скобки и кавычки начинают заслонять саму структуру базы.

SQL-команды можно вынести в отдельный файл с расширением .sql. Такой файл называется SQL-скриптом.

Файл setup.sql

Точка с запятой завершает каждую SQL-команду. Один скрипт может содержать сколько угодно команд, а переносы строк и отступы делают его удобным для чтения человеком.

Читаем скрипт целиком

Метод read_all() из библиотеки file возвращает всё содержимое файла одной строкой.

Поэтому в проекте должны лежать рядом два файла: программа main.idyl и скрипт setup.sql.

Выполняем несколько команд

Обычный execute() принимает ровно одну SQL-команду. exec_script() предназначен для текста с несколькими командами. Если какая-либо команда ошибочна, выполнение остановится на ней.

Метод exec_script() не создаёт транзакцию автоматически. Если скрипт должен выполниться строго целиком, оберните его в уже знакомые begin_transaction(), commit() и rollback().

Что происходит с SELECT внутри скрипта?

exec_script() ничего не возвращает. Если внутри скрипта есть SELECT, найденные им строки будут отброшены. Для чтения данных выполняйте отдельный execute() и сохраняйте полученный sqlite.Result.
