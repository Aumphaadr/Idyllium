Апостроф внутри фамилии преждевременно закроет SQL-строку. Специально исправлять кавычки вручную тоже не надо: у SQLite уже есть безопасный механизм параметров.

Готовим запрос с пустыми местами

Метод prepare() создаёт подготовленный запрос. Места для данных обозначаются двоеточием и понятным именем: :name, :level, :active.

Двоеточие является частью SQL-записи параметра. Когда мы связываем параметр со значением через bind(), имя передаётся уже без двоеточия.

Подставляем значения

bind() принимает строки, числа, булевы значения и null. Значения передаются SQLite отдельно от текста команды, поэтому фамилия с апострофом останется фамилией и не превратится в кусочек SQL.

Закрываем результат и запрос

Даже у команды без строк для чтения есть результат: в нём, например, хранится количество изменённых строк. После выполнения оба объекта следует закрыть.

Типизированные варианты bind

В большинстве программ достаточно обычного bind(). Если нужно явно потребовать определённый SQL-тип, существуют методы bind_int(), bind_int64(), bind_float(), bind_string(), bind_bool() и bind_null().

Это не разные способы выполнить запрос. Они только точнее описывают тип подставляемого значения.
