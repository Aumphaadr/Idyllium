Урок 23: Работа с файлами

Как сохранять данные между запусками программы и читать внешние тексты

Зачем нужны файлы?

До сих пор все наши программы работали только с данными, введёнными вручную или сгенерированными на лету. Но реальные программы часто:

  • сохраняют результаты (например, очки в игре);
  • читают внешние данные (например, список слов для викторины);
  • обмениваются информацией с другими программами.

Для этого нужен файл — постоянное хранилище на диске.

Библиотека file

В Idyllium вся работа с файлами сосредоточена в библиотеке file. Она предоставляет функции и типы для записи и чтения.

Основные типы:

  • file.ostream — поток для записи в файл;
  • file.istream — поток для чтения из файла.

Запись данных в файл

Чтобы записать данные в файл, нужно:

  1. Открыть файл в режиме "write";
  2. Записать нужные строки;
  3. Закрыть файл с помощью .close().
use file;

main() {
    file.ostream fout = file.open("output.txt", "write");
    fout.write_line("Привет, мир!", '\n');
    fout.write_line("Сейчас ", 2025, " год", '\n');
    fout.write_line("Файл создан с помощью Idyllium!");
    fout.close();
}

После выполнения программы в той же папке появится файл output.txt со следующим содержимым:

Привет, мир! Сейчас 2025 год Файл создан с помощью Idyllium!

Функция write_line() работает так же, как console.write() — принимает любое количество аргументов и объединяет их в строку.

Чтение данных из файла

Чтобы прочитать файл, нужно:

  1. Открыть файл в режиме "read";
  2. Пока есть строки — читать их по одной;
  3. Закрыть файл.
use console;
use file;

main() {
    file.istream fin = file.open("input.txt", "read");
    while (fin.has_next_line())
    {
        string one_line = fin.read_line();
        string fixed_line = one_line.simplify(); // удаляет '\n' и другие управляющие символы
        console.write(fixed_line, '\n');
    }
    fin.close();
}

Допустим, у нас есть файл input.txt:

Яблоко Банан Апельсин

Программа выведет в консоль:

Яблоко Банан Апельсин

Метод .simplify() удаляет символы перевода строки и других управляющих последовательностей, оставляя «чистый» текст.

Важные правила

  • Файл открывается с указанием режима: "write" или "read";
  • После работы файл обязательно нужно закрывать с помощью .close() — иначе данные могут не сохраниться или повредиться;
  • Если попытаться открыть несуществующий файл в режиме "read" — программа выдаст ошибку;
  • Если открыть существующий файл в режиме "write" — его предыдущее содержимое будет полностью стёрто.

Это не ограничения — это защита от случайных потерь данных.

Практическое применение

Файлы позволяют:

  • Сохранять настройки игры между сессиями;
  • Загружать словари, базы данных, уровни;
  • Экспортировать результаты работы программы в удобном формате (например, CSV);
  • Обеспечивать переносимость данных — файл можно отправить другу или открыть в текстовом редакторе.

Теперь ваши программы — не просто временные скрипты, а полноценные инструменты.
