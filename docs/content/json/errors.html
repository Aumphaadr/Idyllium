Сначала вспомним правила JSON-текста

Большинство ошибок JSON не злые, а механические. Формат строгий, поэтому важно помнить несколько правил:

  • ключи объекта пишутся в двойных кавычках;
  • строки тоже пишутся в двойных кавычках;
  • объекты заключаются в фигурные скобки { };
  • массивы заключаются в квадратные скобки [ ];
  • между элементами стоят запятые, но после последнего элемента запятая не нужна;
  • комментариев в JSON нет.

Невалидный JSON

Перед чтением файла можно вызвать json.is_valid(). Это удобно, если вы хотите показать своё сообщение вместо ошибки парсинга.

Файл сохранения повреждён

Комментарии в JSON запрещены

В Idyllium комментарии можно писать через //, но в JSON-файле так делать нельзя. JSON — это формат данных, а не файл с программным кодом.

Такой файл не распарсится. Если нужен комментарий для человека, лучше объяснить данные в документации проекта или добавить обычное поле вроде "description", если это действительно часть данных.

Несуществующий ключ

main.idyl:5: runtime error: json object has no key 'level'

Неподходящий тип

main.idyl:5: runtime error: json value is string, expected int

Повтор ключа и неверный индекс

add() не даст создать второй такой же ключ, а at() не даст выйти за границы массива.
