Урок 23: Работа с файлами

Как сохранять данные между запусками программы и читать внешние файлы

Зачем работать с файлами?

Пока мы выводили данные только в консоль — и после завершения программы они исчезали.

Файлы позволяют:

  • сохранять результаты работы программы;
  • загружать настройки или данные извне;
  • обмениваться информацией с другими программами.

В Idyllium для этого есть библиотека file.

Запись данных в файл

Чтобы записать что-то в файл, нужно:

  1. Открыть файл в режиме "write";
  2. Записать строки с помощью метода .write_line();
  3. В конце — обязательно закрыть файл методом .close().
use file;

main() {
    file.ostream fout = file.open("output.txt", "write");
    fout.write_line("Привет, мир!", '\n');
    fout.write_line("Сейчас ", 2025, " год", '\n');
    fout.write_line("Файл создан с помощью Idyllium!");
    fout.close();
}

После выполнения программы в той же папке появится файл output.txt со следующим содержимым:

Привет, мир! Сейчас 2025 год Файл создан с помощью Idyllium!

Метод .write_line() работает как console.write() — принимает любое количество аргументов и не делает автоматический перенос строки.

Чтение данных из файла

Чтобы прочитать файл, нужно:

  1. Открыть его в режиме "read";
  2. Пока есть строки — читать их по одной с помощью .read_line();
  3. Закрыть файл после завершения.
use console;
use file;

main() {
    file.istream fin = file.open("input.txt", "read");
    while (fin.has_next_line())
    {
        string one_line = fin.read_line();
        string fixed_line = one_line.simplify(); // убирает '\n' и другие управляющие символы
        console.write(fixed_line, '\n');
    }
    fin.close();
}

Как выглядит исходный файл?

Предположим, файл input.txt содержит:

Кот спит. Рыба плавает. Луна светит.

Тогда программа выведет в консоль:

Кот спит. Рыба плавает. Луна светит.

Без метода .simplify() каждая строка включала бы символ '\n' в конце — и при выводе появлялись бы лишние пустые строки.

Важные детали

  • Файл открывается относительно папки с программой — если main.idyl лежит в Projects/, то и файлы читаются/пишутся туда же.
  • Если при чтении файла его не окажется — Idyllium выдаст ошибку (в отличие от некоторых языков, которые молча вернут пустоту).
  • Метод .close() обязателен: он гарантирует, что все данные действительно запишутся на диск, а ресурсы освободятся.

Это не формальность — это защита от потери данных.

Итог

Работа с файлами в Idyllium:

  • проста и предсказуема;
  • требует явного открытия и закрытия;
  • не позволяет случайно повредить данные;
  • не скрывает ошибки — они становятся видимыми сразу.

Теперь ваши программы могут «помнить» что-то и взаимодействовать с внешним миром.
