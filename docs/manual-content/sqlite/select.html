Просим базу вернуть данные

Команда SELECT выбирает данные из таблицы. После слова SELECT перечисляются нужные колонки, а после FROM — таблица.

Результат может содержать ноль, одну или много строк, поэтому execute() не может сразу вернуть одно имя. Он возвращает объект sqlite.Result — набор найденных строк.

Первая строка не выбирается сама

Новый Result находится перед первой строкой. Метод next() переходит к следующей строке и отвечает, удалось ли это сделать.

Если таблица пуста, next() вернёт false, а тело if не выполнится. Вызывать методы чтения до успешного next() нельзя: у результата ещё нет текущей строки.

Читаем все строки

Для неизвестного заранее количества строк отлично подходит цикл while. Каждый вызов next() передвигается дальше, а после последней строки возвращает false.

Liam: level 7 Mira: level 12 Nika: level 4

Каждому типу — свой метод чтения

Idyllium не превращает данные базы во что попало. Текст читается через get_string(), целое число — через get_int(), дробное — через get_float(), а булево — через get_bool(). Во все методы передаётся название колонки.

В самой SQLite отдельного булева типа нет. Для него используются числа 0 и 1, а get_bool() аккуратно превращает их в false и true.

Порядок и вычисляемая колонка

ORDER BY level DESC ставит игроков с большим уровнем первыми. SQL умеет вычислить значение, а AS даёт вычисленной колонке понятное имя.
