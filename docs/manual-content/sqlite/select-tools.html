Когда колонок стало много

До сих пор после SELECT мы перечисляли нужные колонки. Звёздочка означает «все колонки таблицы» и удобна, когда хочется быстро осмотреть данные.

Таблица players

Для исследования базы это удобно. В готовой программе всё же полезнее перечислять только действительно нужные колонки: так результат понятнее, а изменение таблицы не принесёт программе неожиданные данные.

Только разные значения

В таблице пять игроков, но классов персонажей всего три. Слово DISTINCT убирает повторяющиеся строки результата.

Результат DISTINCT
class_name
archer
mage
warrior

База умеет считать

Функции COUNT, MIN, MAX и AVG собирают много строк в одну итоговую строку. COUNT(*) считает все найденные строки, а остальные функции работают с указанной колонкой.

Краткая статистика по всем игрокам
player_countmin_levelmax_levelaverage_level
54127.6

Статистика по группам

GROUP BY class_name сначала собирает игроков с одинаковым классом в группы, а затем запускает функции отдельно для каждой группы.

Количество и средний уровень по классам

Для Idyllium это всё ещё обычный результат. Если запрос возвращает одну итоговую строку, достаточно одного успешного next().
