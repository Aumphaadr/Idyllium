Шрифт — тоже ресурс

Картинку мы сначала загружаем, а затем помещаем в gui.ImageBox. Со шрифтом логика похожая: сначала создаём объект fonts.Font, загружаем в него файл, а затем назначаем этот объект виджету.

Отдельно запоминать классы для TTF, OTF и WOFF не нужно. Один класс Font сам определит формат по содержимому файла.

Свойства src, format и is_loaded можно читать, но нельзя подменять вручную. За их правдивость отвечает Idyllium.

Назначаем шрифт надписи

Сам файл шрифта не хранит размер текста. Один и тот же шрифт может быть маленьким в подписи и большим в заголовке. Поэтому font отвечает за рисунок букв, а font_size остаётся свойством виджета.

Один раз назначить окну

Если надписей и кнопок много, повторять font = lobster у каждого виджета быстро надоест. Шрифт можно назначить окну: тогда его унаследуют все дочерние виджеты, которые не выбрали собственный шрифт.

Та же механика работает у gui.Frame. Шрифт фрейма передаётся его потомкам и может заменить шрифт окна только внутри этого фрейма. Это похоже на уже знакомое наследование text_color.

Если в файле чего-то нет

Если путь неверный или файл на самом деле не является поддерживаемым шрифтом, программа завершится с понятной runtime error. Переименование случайного файла в .ttf шрифтом его не сделает.

Бывает и другая ситуация: шрифт настоящий, но в нём нет некоторых русских букв или специальных символов. Тогда только отсутствующие символы будут показаны запасным системным шрифтом. Поэтому перед уроком стоит проверить, что выбранный файл действительно содержит кириллицу.
