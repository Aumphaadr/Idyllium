Когда объектов становится много

Пока объект один, его свойства нетрудно заполнить вручную. Но представим игру с тремя героями:

На каждого героя приходится одна и та же пачка присваиваний. Если у героя появятся мана, уровень и инвентарь, пачка станет ещё длиннее. Хочется один раз описать правильное создание героя, а затем только передавать данные.

Конструктор

Конструктор — особый метод, который запускается при создании объекта. Его имя совпадает с именем класса:

Kaspar: 500

Выражение Hero kaspar("Kaspar", 500) создаёт объект, после чего конструктор раскладывает аргументы по его полям. Обращение к полям внутри конструктора, как и внутри обычного метода, требует this..

Параметры без значения по умолчанию идут первыми. Поэтому имя обязательно, а здоровье можно не указывать.

Свежий объект прямо внутри выражения

Иногда имя для промежуточной переменной только мешает. Имя класса можно вызвать как функцию: такой вызов создаст и вернёт новый объект.

Каждый Hero(...) создаёт отдельного героя. Именованные аргументы можно переставлять местами. Ключевое слово new не нужно: его в синтаксисе Idyllium нет.

Объект внутри объекта

Конструктор-выражение работает и в поле класса. Это удобно для композиции: герой сразу получает собственный объект здоровья.

При создании двух героев каждый получит собственный Health. Изменение здоровья одного героя не меняет здоровье другого.

Что запомнить

  • Конструктор описывает начальное состояние нового объекта.
  • Форма Hero hero(...) создаёт именованную переменную.
  • Форма Hero(...) возвращает свежий объект прямо в выражение.
  • Значения по умолчанию и именованные аргументы работают в обеих формах.
  • У класса без явного конструктора допустим вызов Empty(), но передавать ему аргументы нельзя.
