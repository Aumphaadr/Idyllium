Зачем вспоминать файлы?

JSON почти всегда появляется рядом с файлами. Если программа умеет сохранить данные в файл, эти данные переживут закрытие программы, перезапуск компьютера и возвращение ученика к проекту через неделю.

В игре с системой сохранений без файлов не обойтись: нужно помнить имя игрока, уровень, деньги, здоровье, предметы в инвентаре и много других вещей. Начнём с простого повторения.

В файле save.txt окажутся две строки: имя игрока и уровень. Это уже похоже на сохранение игры, только очень маленькое.

Прочитаем сохранение обратно

Теперь откроем тот же файл на чтение. Имя читается как строка, а уровень тоже сначала приходит строкой, поэтому его нужно превратить в число.

Liam имеет уровень 7

Когда данных становится больше

Представим многопользовательскую игру. У каждого игрока есть имя и уровень. Можно хранить имена в одном файле, а уровни в другом.

Для двух параметров такой подход ещё можно терпеть. Но если у игрока появятся пароль, деньги, здоровье, мана и инвентарь, придётся заводить всё больше файлов и массивов. Код станет похож на шкаф, где каждый ящик подписан отдельно, но связаны эти ящики только нашей памятью.

Зачем тут JSON

JSON позволяет хранить сложные данные одним цельным объектом: имя рядом с уровнем, деньги рядом со здоровьем, а инвентарь прямо внутри игрока.

В следующих уроках мы научимся создавать такие JSON-данные, записывать их в файл и читать обратно без десятка разрозненных массивов.
