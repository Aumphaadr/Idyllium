Библиотека encoding

Как работать с кодировками текста — превращать символы в числа и обратно

Символ, номер и байты

Компьютер хранит данные числами. Чтобы буквы разных языков не спорили за номера, существует таблица Unicode. В ней у каждого символа есть постоянный номер, который называется кодовой точкой.

Например, кодовая точка русской буквы 'Я' всегда равна 1071. Она не меняется от того, куда мы собираемся записать букву.

А вот файл состоит не из кодовых точек, а из маленьких чисел от 0 до 255, то есть из байтов. Кодировка отвечает на вопрос: «Какими байтами записать этот текст?»

Что рассматриваемРезультат для 'Я'
Кодовая точка Unicode1071
Байт Windows-1251[223]
Байт KOI8-R[241]
Байты UTF-8[208, 175]

Вот откуда берутся разные числа. 1071 описывает сам символ, а остальные значения описывают его запись в конкретной кодировке.

Поддерживаемые кодировки

Idyllium поддерживает четыре кодировки:

  • utf-8 — современная универсальная кодировка, способная хранить все символы Unicode;
  • ascii — маленькая кодировка с латиницей, цифрами и основными знаками, использующая значения от 0 до 127;
  • windows-1251 — однобайтовая кириллическая кодировка из старых Windows-программ;
  • koi8-r — ещё одна однобайтовая кириллическая кодировка, известная по Unix-системам.

Список можно получить и прямо из программы:

ascii utf-8 windows-1251 koi8-r

Кодовая точка символа

Функция encoding.char_to_codepoint() получает один символ и возвращает его постоянный номер Unicode. Имя кодировки ей не нужно.

65 1071

Обратную работу выполняет encoding.codepoint_to_char():

Я

Превращаем текст в байты

Функция encoding.encode(текст, кодировка) превращает строку в динамический массив байтов. Теперь кодировка действительно имеет значение:

UTF-8: [208, 175] Windows-1251: [223] KOI8-R: [241]

В UTF-8 русской букве понадобилось два байта, а в старых кириллических кодировках хватило одного. Зато UTF-8 умеет хранить не только русский и английский, а весь Unicode.

Собираем текст из байтов

encoding.decode(байты, кодировка) выполняет обратное преобразование. Важно указать ту же кодировку, которой пользовались при записи.

Байты: [207, 240, 232, 226, 229, 242] Текст: Привет

Если прочитать старый файл не той кодировкой, числа будут истолкованы иначе. Так и появляются знаменитые «кракозябры».

Ошибки не прячутся

ASCII не знает русских букв. Idyllium не станет молча заменять неизвестный символ вопросительным знаком:

main.idyl:1: runtime error: character 'Ю' is not valid ASCII at position 0

Повреждённые байты UTF-8 тоже дают ошибку. Например, байт 208 начинает двухбайтовый символ, но в массиве нет его продолжения:

main.idyl:2: runtime error: encoding.decode() invalid UTF-8 at byte 0 (0xD0): incomplete sequence

Наконец, имя кодировки тоже проверяется:

main.idyl:1: runtime error: unknown encoding 'cp866'

Практическое применение: шифр Цезаря

Кодовые точки позволяют выполнять с буквами простую математику. Сдвинем каждую латинскую букву на три позиции:

Исходное: Hello World Зашифрованное: Khoor Zruog

Итог

  • encoding.char_to_codepoint(символ) — постоянная кодовая точка Unicode;
  • encoding.codepoint_to_char(число) — символ по кодовой точке;
  • encoding.encode(строка, кодировка) — строка в массив байтов;
  • encoding.decode(байты, кодировка) — массив байтов в строку;
  • encoding.list_encodings() — список поддерживаемых кодировок.

Запомните главное: кодовая точка описывает символ, а кодировка описывает его запись в байтах. Это похожие, но не одинаковые вещи.
